git config --global user.name ""

Komenda pozwala ustawić nazwę użytkownika Git. --global sprawia że Git użyje tych informacji do wszystkich rzeczy wykonywanych na systemie

git add .

Komenda dodaje wszystkie pliki znajdujące się w obecnym katalogu i wszystkich podkatalogach do plików śledzonych przez git-a

gitk --all

gitk jest to gui wbudowane w git'a, które wyświetla zmiany dokonane w repozytorium poprzez graf commit'ów oraz wyświetla informacje dotyczące każdego commit'a. Flaga --all daje pełnieszy wgląd w repozytorium, poprzez pokazywanie innych branch'ów, nie tylko tego w którym się znajdujemy oraz tagów itp.

git diff --staged

Pokazuje dokładne zmiany pomiedzy twoim indexem, a ostatnim commitem. Komenda służy do sprawdzenia, jakie zmiany zajdą w repozytorium, jeżeli użyjemy komendy git commit.

git branch "name"

Komenda tworzy brancha o wpisanej nazwie

git branch -d "branch"

Komenda usuwa brancha o podanej nazwie

git commit -a -m ""

Komenda commituje zmiany. To znaczy w praktyce tyle ze zmiany zostaja wprowadzone do bazy danych z komentarzem wpisanym po -m. Znacznik -a natomiast oznacza tyle ze commitujemy tylko pliki zmodyfikowane lub usuniete, natomiast nowe pliki nie beda commitowane

git branch --merged

Komenda ta, w odroznieniu od zwyklego 'git branch' pokazuje tylko branche ktore zostaly wczesniej zmergowane (tzn. inaczej mowiac polaczone) z glownym branchem, ktory zazwyczaj nazywa sie master.
